# 环境构建

* graalvm-community-jdk-21 版本
* MySQL Server 8.0
* redis 5.X
* maven 3.9.5
* nodejs v20.11.1
* npm 10.2.4

## 后端部署

### 导入SQL

将script目录下的.sql文件导入数据库，再按顺序导入update目录下的.sql文件

### 配置文件修改

在后端项目中的FMS-admin/src/main/resources/application-dev.yml文件中
修改#数据库配置，将数据库源的url和访问用户名密码修改
修改redis配置中的源url和访问用户名密码

### 后端运行

运行FMS-admin/src/main/java/org/dromara/DromaraApplication.java中的DromaraApplication类

## 前端运行

```bash

# 安装依赖
npm install --registry=https://registry.npmmirror.com

# 启动服务
npm run dev

# 构建生产环境 
npm run build:prod

# 前端访问地址 http://localhost:80
```

# 操作说明

1. 登录注册

* 访问前端IP地址即可进入系统的登录注册页。完成验证码挑战并输入用户名、密码后点击登录按钮即可进入系统。
* 点击登录页中的立即注册链接进入注册页面。在注册页面需要完成验证码挑战，输入符合长度等限制规范的用户名和密码后点击注册即可完成用户的注册。
* 进入主页后，点击左侧菜单即可开启对应功能页面。在右上角用户头像点击个人中心即可进入个人信息页面修改用户的基本信息；通过布局设置可以修改主页布局，如界面颜色；点击退出登录会返回登录页并退出当前账户。

2. 系统管理

* 用户管理
用户点击主页左边的系统管理栏，再点击用户管理。在页面左侧可以根据岗位部门筛选用户，页面上方也提供了根据用户名、用户状态等条件筛选用户的方法。还可以代替注册页面直接增加用户，点击用户表单中的新增按钮，新增表单就会被显示。键入要新增的用户信息，点击确认后即可添加用户，添加时也可直接对用户分配部门岗位以及角色。勾选单个用户后点击上方修改按钮（勾选多个用户后按钮会变不可用），或直接点击下面展示表格的编辑icon图标即会显示修改用户表单。修改信息后点击确定，系统提示成功消息即代表信息修改成功。

3. 商户管理
用户点击右边商户管理栏再点击商户管理即可打开商户管理页面。页面上方提供了根据商户id、负责人、商户等级、商户地址、商户状态的条件搜索。点击展示表格上方的新增按钮会打开新增商户表单，该表单与点击修改商户表单使用了同一个前端组件，填入信息后系统会根据用户操作自动判断新增或修改商户操作。

4. 订单管理

* 商品管理
在商品管理页面，与之前的页面相同，页面上方都提供了条件搜索，帮助用户更好的筛选商品信息。商品新增和修改用了同一个组件，点击新增按钮和对应商品操作列的编辑按钮即可呼出对应的表单。

* 订单管理
在订单管理页面，加盟商户只能查看和新增订单，品牌市场经理才具有订单的修改权限。在页面上部同样配备了条件搜索器。点击新增按钮和下方表格的编辑icon图标，即可呼出新增或修改订单信息的表单。点击下方表格末列的删除icon图标会弹出确认删除的消息框。
